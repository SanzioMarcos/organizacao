Guia de Início Rápido: Convenções Concretas

Os princípios da indústria aplicados na prática de projetos profissionais.

Índice Rápido

Commits (Conventional Commits)

Sempre comece com uma flag:

feat: adiciona autenticação JWT
fix: corrige timeout da API
docs: atualiza README
test: adiciona testes do usuário
refactor: reorganiza camada de serviços
chore: atualiza dependências
ci: adiciona workflow GitHub Actions

Regra:

<tipo>: descrição corta

Tipos mínimos: feat, fix, docs, test, refactor, chore, ci.

Estrutura de pastas Python

project/
├── src/
├── tests/
├── docs/
├── .github/
├── Dockerfile
├── requirements.txt
└── README.md

Se for API:

src/
├── api/
├── services/
├── models/
├── database/
└── utils/

README

Ordem padrão:

# Nome do Projeto

Descrição

Instalação

Uso

API

Testes

Docker

Licença

Tratamento de erros

Nunca:

except:
    pass

Sempre:

except Exception as e:
    logger.error(str(e))
    raise

Testes

Espelhe a estrutura do código.

src/
└── services/
    └── user.py

tests/
└── services/
    └── test_user.py

Nome padrão:

test_<arquivo>.py

Funções:

def test_create_user():

API REST

URLs sempre no plural:

/users
/products
/orders

Métodos:

GET | POST | PUT | DELETE

Documentação:

Swagger/OpenAPI

Hoje isso é praticamente obrigatório.

Docker

Arquivo obrigatório: Dockerfile

Opcional: docker-compose.yml

Comando padrão:

docker build .
docker run .

CI/CD

Pasta padrão: .github/workflows/

Arquivo: ci.yml

Fluxo mínimo:

Checkout -> Instalar dependências -> Executar testes -> Executar lint

Engenharia de IA

Além das regras acima:

models/
prompts/
rag/
evaluations/

E sempre: .env.example

Nunca: .env

10 Regras Obrigatórias para o GitHub

Se reduzirmos tudo a regras cruciais para qualquer projeto profissional de backend, MLOps e engenharia de IA:

  1. Use Conventional Commits.
  2. Código dentro de src/.
  3. Testes dentro de tests/.
  4. README com instalação e uso.
  5. Dockerfile funcional.
  6. Workflow CI automático.
  7. Nunca use except: pass.
  8. APIs documentadas com OpenAPI/Swagger.
  9. Variáveis confidenciais mapeadas em .env.example.
  10. Um commit, uma responsabilidade.
